## Documentation Organization

### By Use Case

**Setting up Ruff in a new project:**
1. Start with `installation.md`
2. Review `configuration.md` for setup
3. Check `integrations.md` for your workflow
4. See `editors/` for your specific editor

**Configuring Ruff behavior:**
1. Review `settings.md` for all options
2. Check `configuration.md` for configuration patterns
3. Review `linter.md` for linter-specific settings
4. See `formatter/overview.md` for formatter settings

**Understanding and fixing specific rule violations:**
1. Start with `rules/rules.md` for overview
2. Find specific rule in `rules/` directory by rule name
3. Each rule file contains description, examples, and fix guidance

**Formatting code:**
1. Review `formatter/overview.md` for formatter basics
2. Check `formatter/black.md` if migrating from Black
3. See `configuration.md` for formatter settings

**Editor integration:**
1. Start with `editors/overview.md`
2. Follow `editors/setup.md` for installation
3. Review `editors/features.md` for capabilities
4. Configure with `editors/settings.md`
